# Notikeep — пошаговый план ваших действий до публикации в RuStore

Это то, что **должны сделать вы** (руками, вне кода) — с объяснением, зачем каждый
шаг. Код готов: мультивыбор, реклама РСЯ (на демо-блоке), release-конфиг, черновик
политики. Порядок важен: шаги идут так, чтобы не переделывать.

Обозначения: ⏱️ ≈ время, 🔑 нужен аккаунт/доступ, ⚠️ критично.

---

## ШАГ 1. Рекламная сеть Яндекса (РСЯ) — получить боевой blockId ⏱️~1 день (модерация)
**Зачем:** сейчас в release стоит демо-баннер Яндекса. С ним нельзя зарабатывать, и
модерация может придраться. Нужен ваш реальный рекламный блок.

1. 🔑 Зарегистрируйтесь в РСЯ: <https://partner.yandex.ru> → «Рекламная сеть».
2. Добавьте площадку типа «Мобильное приложение» (укажите будущий applicationId
   `com.notikeep` и площадку RuStore).
3. Создайте рекламный блок «Баннер». После модерации получите `blockId` вида
   `R-M-XXXXXX-Y`.
4. Впишите его в `gradle.properties` (в корне проекта, **не коммитьте с реальным
   ключом**, либо через CI-секрет):
   ```
   notikeep.ads.bannerId=R-M-XXXXXX-Y
   ```
   Код уже читает это свойство в release; без него остаётся демо-блок.

**Проверка:** соберите release и убедитесь, что баннер загружается (см. ШАГ 6).

---

## ШАГ 2. Release keystore — сгенерировать и забэкапить ⚠️ ⏱️15 мин
**Зачем:** RuStore принимает только подписанные сборки. Ключ подписи **навсегда**
привязан к приложению — потеряете ключ, не сможете выпускать обновления. Бэкап
обязателен.

1. Сгенерируйте ключ (в корне проекта):
   ```
   keytool -genkey -v -keystore notikeep-release.jks -keyalg RSA -keysize 2048 \
           -validity 10000 -alias notikeep
   ```
   Задайте надёжные пароли, запомните alias `notikeep`.
2. Скопируйте шаблон и заполните:
   ```
   cp keystore.properties.example keystore.properties
   ```
   Впишите путь к `.jks`, пароли, alias. Файл уже в `.gitignore` — в гит не попадёт.
3. ⚠️ **Забэкапьте `notikeep-release.jks` и пароли в ДВУХ разных местах** (например,
   менеджер паролей + зашифрованный офлайн-носитель). Без этого файла обновления
   невозможны.

**Проверка:** `./gradlew bundleRelease` — в логе `signReleaseBundle` подпишет вашим
ключом (а не debug).

---

## ШАГ 3. Политика конфиденциальности — разместить по URL ⚠️ ⏱️30 мин
**Зачем:** при разрешениях «доступ к уведомлениям» и «статистика использования»
RuStore **требует** политику конфиденциальности по публичной ссылке. Она же нужна
для рекламы и аналитики.

1. Откройте черновик `PRIVACY_POLICY.md`, заполните: ФИО/наименование, контактный
   e-mail, дату. Удалите пункт про push, если push пока нет.
2. Разместите текст по публичному URL. Самое простое бесплатно:
   - GitHub Pages (создать репозиторий, включить Pages), или
   - Telegraph (telegra.ph), или ваш сайт.
3. Сохраните URL — понадобится в карточке RuStore.

## ШАГ 4. Финальная проверка кода перед сборкой ⏱️10 мин
**Зачем:** убедиться, что release-сборка с обфускацией не падает и тесты зелёные.

Выполните:
```
./gradlew testReleaseUnitTest lintRelease
```
Оба должны пройти (у меня проходят). Если `lintRelease` ругается — смотрите отчёт
`app/build/reports/lint-results-release.html`.

---

## ШАГ 5. Собрать AAB для загрузки ⏱️5 мин
**Зачем:** RuStore принимает Android App Bundle (`.aab`).

```
./gradlew bundleRelease
```
Готовый файл: `app/build/outputs/bundle/release/app-release.aab`.
Убедитесь, что он подписан ВАШИМ ключом (ШАГ 2 выполнен).

---

## ШАГ 6. Тест на реальном устройстве ⚠️ ⏱️30–60 мин
**Зачем:** R8/обфускация и рантайм-разрешения ведут себя иначе, чем в debug.
Эмулятор не покажет часть проблем (листенер, батарея).

Установите release-сборку на телефон:
```
./gradlew installRelease
```
Проверьте по чек-листу:
- [ ] Онбординг: запрос доступа к уведомлениям, запрос POST_NOTIFICATIONS (Android 13+).
- [ ] Уведомления реально сохраняются в архив.
- [ ] Мультивыбор в «Приложениях» → массовая смена правила работает.
- [ ] Мультивыбор в уведомлениях → удаление выбранных работает.
- [ ] Сводное уведомление появляется с иконкой-щитом (не серый квадрат).
- [ ] Реклама-баннер грузится (при боевом blockId) и **не мешает** списку.
- [ ] **Оффлайн** (выключить интернет): приложение не падает, баннер просто скрыт.
- [ ] Перезагрузить телефон → листенер восстанавливается, сохранение продолжается.
- [ ] Отключить аналитику в настройках → баннер/аналитика уважают отказ.

## ШАГ 7. Карточка приложения в RuStore ⏱️1–2 часа
**Зачем:** без заполненной карточки и деклараций модерация отклонит.

1. 🔑 Зарегистрируйтесь как разработчик: <https://console.rustore.ru>.
2. Создайте приложение, applicationId `com.notikeep` (⚠️ **навсегда**, менять нельзя).
3. Загрузите AAB (из ШАГ 5).
4. Заполните:
   - **Иконка** (уже есть в проекте).
   - **Скриншоты** — минимум несколько: архив, экран приложения, «Приложения» с
     мультивыбором, настройки. Можно с подписями-фичами.
   - **Описание (RU)** — что делает приложение и **зачем каждое разрешение**
     (особенно доступ к уведомлениям и статистику использования — модерация смотрит).
   - **Ссылка на политику конфиденциальности** (ШАГ 3).
   - **Декларация данных/разрешений** — заполнить честно: уведомления хранятся
     локально, аналитика AppMetrica, реклама РСЯ.
   - **Возрастной рейтинг** — пройти анкету.
   - **Категория**, **«что нового»** (changelog для 1.0.0).

## ШАГ 8. Отправить на модерацию ⏱️до нескольких дней
**Зачем:** финал. После одобрения приложение публикуется.

- Проверьте, что versionCode=1, versionName=1.0.0.
- Отправьте на проверку. Следите за статусом; при отклонении читайте причину и
  правьте (чаще всего — обоснование разрешений или политика).

## ПОЗЖЕ (после первого релиза, необязательно для MVP)

### Push-уведомления (RuStore Push SDK)
**Зачем:** рассылки пользователям («новая версия», акции) из консоли RuStore без
своего сервера.
1. Подключить `ru.rustore.sdk:pushclient`, получить push-токен.
2. Создать проект в консоли RuStore Push, слать кампании вручную.
3. `POST_NOTIFICATIONS` уже запрашивается — переиспользуется.
4. Токены хранит RuStore (свой сервер не нужен).

### Рекомендации из аудита (не блокеры)
- Подсветить в онбординге статус «доступ выдан ✓».
- Будильник на смену суток для точного сброса счётчика дневной сводки.
- Заменить демо-баннер РСЯ на боевой (ШАГ 1) — **обязательно** до релиза.
