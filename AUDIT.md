## Безопасность / приватность

### ✅ FIXED — debug-логи аналитики в release
`AppMetricaAnalytics.track` логировал каждое событие через `Log.d` всегда.
Обёрнуто в `if (BuildConfig.DEBUG)` — в release логов нет. Содержимого уведомлений
в логах не было и раньше (только имена событий), так что утечки данных не было.

### ⚠️ Демо-баннер РСЯ в release
`ADS_BANNER_UNIT_ID` в release падает на `demo-banner-yandex`, пока не задан
`notikeep.ads.bannerId`. Публикация с демо-блоком = нет дохода + потенциально
претензия модерации. **Действие:** задать боевой blockId перед сборкой релиза
(см. ACTION_PLAN.md).

### OK — приватность уведомлений
Содержимое уведомлений хранится только локально (Room), никуда не отправляется.
Аналитика/реклама получают только обезличенные данные. Согласие связано с тумблером.
Проверено: `NotificationRepositoryImpl`, `AppMetricaAnalytics`, `AdsInitializer`.

### OK — обработка разрешений
Все системные вызовы (`SystemSettings`, `ListenerRebinder`) обёрнуты либо ведут в
системные экраны. `POST_NOTIFICATIONS` запрашивается в рантайме, отказ обрабатывается
тихо (`DailySummaryNotifier.canPost()` + try/catch на `notify`).

## Производительность

### ✅ FIXED — иконка удалённого приложения роняла загрузку
`AppIconFetcher.fetch()` вызывал `getApplicationIcon`, который бросает
`NameNotFoundException` для удалённых приложений (а такие строки есть в архиве).
Теперь fallback на `defaultActivityIcon` — не крашится, не спамит лог.

### ✅ FIXED — двойной расчёт атрибутов события
`AppMetricaAnalytics.track` вызывал `attributes(event)` дважды. Вычисляется один раз.

### OK — крупные списки
Детальный экран и поиск переведены на Paging 3 (память постоянна). Сводки архива
агрегируются в SQLite. Индексы БД (миграция v6) ускоряют антидубль и `sbnKey`.
Дедуп использует `EXISTS` вместо `COUNT`. Иконки грузятся через Coil off-main-thread
с memory-cache. `InstalledAppsProvider` кэширует список и делает один IPC-запрос.

### Наблюдение — DailySummaryController и полночь
Граница «начало дня» вычисляется при подписке и пересчитывается только при
переключении тумблера. При работе через полночь счётчик за сегодня может быть
неточным до перезапуска/переключения. Не критично для realtime-режима; при желании
позже добавить будильник на смену суток. (Задокументировано в коде.)

## Ошибки пользователя / UX

### ⚠️ Онбординг можно завершить без главного разрешения
Кнопка «Готово» доступна, даже если пользователь не выдал доступ к уведомлениям.
Тогда приложение открывается с пустым архивом. Экран настроек и пустое состояние
это объясняют (honest empty-state есть), но онбординг мог бы **подсвечивать
статус выдачи** (галочка «доступ выдан»). Рекомендация, не блокер.

### OK — тихие отказы
Отказ в `POST_NOTIFICATIONS`, отсутствие usage-разрешения, оффлайн-реклама —
всё обрабатывается без крашей (баннер скрывается, usage → сортировка A-Z).

### OK — многовыбор
Оба режима (Приложения, Уведомления) имеют выход по системной «назад» (`BackHandler`),
подтверждение массового удаления, счётчик выбранного.

## Хардкод

### ✅ FIXED — строки онбординга и «Очистить»
Весь текст онбординга и `contentDescription="Очистить"` были inline-русскими
строками. Вынесены в `strings.xml` (`onboarding_*`, `action_clear`). Теперь всё
локализуемо и соответствует паттерну проекта.

### Приемлемо — BRAND_BLUE в DailySummaryNotifier
Цвет `0xFF2E44BE` продублирован из темы. Тема — Compose (`Color`), нотифаер —
data-слой (Android `Int`); переиспользовать Compose-цвет в data-слое = утечка слоёв.
Оставлено как именованная константа со ссылкой на источник — это меньшее зло, чем
тащить presentation в data.

### OK — конфигурируемые значения
Ключи (AppMetrica, ads blockId) идут через BuildConfig/gradle.properties, не хардкод.
Размер страницы Paging, окно дедупа, размер иконки — именованные константы.

## ООП / архитектура

### Отлично — чистая архитектура соблюдена
- Три слоя: `domain` (модели/порты/use-case, без Android) → `data` (Room/DataStore/
  service/ads) → `presentation` (Compose+ViewModel). Зависимости направлены внутрь.
- Все абстракции связаны через `BindingModule` (`@Binds`) — смена реализации
  (аналитика/биллинг/каталог) это одна строка. `FreeEntitlementProvider` —
  готовый пример для будущего RuStore-биллинга.
- Use-case чистые и юнит-тестируемы без эмулятора (`ApplyRuleUseCase`,
  `DeduplicateUseCase` с инъекцией часов).
- Меню вынесено в `NotikeepBottomBar`, диалог удаления переиспользуется, поиск —
  отдельный компонент. Разделение ответственности выдержано.

### Мелочь — дубль startOfToday / логики окна
`DailySummaryController.startOfToday()` и окно дедупа — локальные, не дублируются
по проекту. Ок.
